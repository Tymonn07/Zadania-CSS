Cechy modyfikowania formatowania w
HTML CSS
umożliwia strukturyzację treści na stronach internetowych za pomocą znaczników, takich jak div, p czy ul
 pozwala na modyfikację wyglądu tych elementów poprzez określanie stylów, takich jak kolor, rozmiar czcionki czy odstępy między elementami, co daje projektantom i programistom szerokie możliwości personalizacji i kształtowania interaktywnych doświadczeń dla użytkowników.

Nowe możliwości CSS

  1. Grid Layout: CSS Grid Layout to potężne narzędzie do tworzenia elastycznych i responsywnych układów strony internetowej. Pozwala ono na definiowanie siatki z kolumnami i wierszami, co umożliwia precyzyjne pozycjonowanie elementów.
  2. Flexbox to inny sposób układania elementów w kontenerze, który zapewnia elastyczność i kontrolę nad ich rozmieszczeniem w obrębie tego kontenera. Jest szczególnie użyteczny do tworzenia elastycznych układów interfejsów użytkownika.
  3. Zmienne CSS pozwalają na definiowanie i ponowne użycie wartości w różnych miejscach arkusza stylów. Umożliwiają one tworzenie bardziej elastycznych i łatwiejszych w utrzymaniu stylów.
  4. CSS umożliwia tworzenie płynnych animacji i przejść bez konieczności używania JavaScriptu. Za pomocą zdefiniowanych w CSS kluczowych klatek i przejść można osiągnąć różnorodne efekty wizualne.
  5. Media Queries pozwalają na dostosowywanie stylów do różnych warunków ekranu, takich jak rozmiar ekranu, orientacja urządzenia czy rozdzielczość. Jest to kluczowe narzędzie do tworzenia responsywnych projektów.
  6. CSS Variables to zmienne, które można definiować w arkuszu stylów i wykorzystywać w wielu miejscach. Umożliwiają one dynamiczne dostosowywanie wartości stylów w zależności od różnych czynników.
  7. CSS oferuje różnorodne efekty filtrów, które pozwalają na manipulację obrazem, takie jak rozmycie, nasycenie kolorów czy zmiana jasności. Są one użyteczne do tworzenia interesujących efektów wizualnych.

Wady CSS

  1. Zbyt duża specyficzność selektorów może prowadzić do problemów z dziedziczeniem i nieprzewidywalnymi zmianami wizualnymi.
  2. Brak możliwości zagnieżdżania w CSS może prowadzić do konieczności powtarzania tych samych selektorów, co zwiększa redundancję kodu i utrudnia jego utrzymanie.
  3. błędne obsługiwanie CSS przez różne przeglądarki
  4. Ponieważ CSS ma globalny zasięg, zmiany wprowadzone w jednym miejscu mogą mieć nieoczekiwane konsekwencje w innych częściach aplikacji lub strony internetowej.
